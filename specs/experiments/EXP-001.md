# EXP-001 — OpenSSL TLS Handshake Bundle

Targets three openssl CVEs that all fire on the same trigger: the TLS handshake
made during every outbound HTTPS connection. Both services are exercised — the .NET
service connects to OpenWeatherMap; the Go service connects to Spotify.

---

## CVE Targets

| CVE | Package | Severity | NVD | Mechanism |
|-----|---------|----------|-----|-----------|
| CVE-2024-5535 | openssl | **CRITICAL 9.1** | https://nvd.nist.gov/vuln/detail/CVE-2024-5535 | Buffer overread in `SSL_select_next_proto()` during ALPN negotiation — leaks up to 255 bytes of memory or crashes |
| CVE-2022-3602 | openssl | HIGH 7.5 | https://nvd.nist.gov/vuln/detail/CVE-2022-3602 | Stack buffer overflow in X.509 name constraint checking (email Punycode) during cert verification |
| CVE-2022-3786 | openssl | HIGH 7.5 | https://nvd.nist.gov/vuln/detail/CVE-2022-3786 | Companion to CVE-2022-3602 — same X.509 name constraint path, overflow via `.` character |

### Why bundled
All three CVEs are triggered by the same application behaviour (making an HTTPS
connection), making them a natural single k6 scenario. They represent two distinct
code paths in the same library: ALPN negotiation (CVE-2024-5535) and X.509 cert
verification (CVE-2022-3602 / CVE-2022-3786).

| CVE-2022-3602 | Network | **Low** | None | None | None | None | **High** |
| CVE-2022-3786 | Network | **Low** | None | None | None | None | **High** |

All three: AV:N / AC:L / PR:N / UI:N — remotely exploitable, low complexity, no
privileges, no user interaction. Worst-case: triggered by any HTTPS connection to a
server that returns a malicious cert or negotiates ALPN adversarially.

## CVE Surface in the Call Path

```
k6 virtual user
  └─ POST /playlist/generate (.NET service)
         │
         ├─ TLS handshake → OpenWeatherMap
         │     ├─ ALPN negotiation  ← CVE-2024-5535 fires here
         │     └─ X.509 cert verify ← CVE-2022-3602 / CVE-2022-3786 fire here
         │
         └─ POST /playlist/generate (Go service)
                │
                └─ TLS handshake → Spotify API
                      ├─ ALPN negotiation  ← CVE-2024-5535 fires here
                      └─ X.509 cert verify ← CVE-2022-3602 / CVE-2022-3786 fire here
```

Every virtual user in k6 that hits `POST /playlist/generate` exercises all three CVEs
across **both** images in a single request.

## Notes

- CVE-2022-3602 and CVE-2022-3786 were the pair that prompted emergency patches in
  OpenSSL 3.0.7 (November 2022). They are among the most widely patched openssl CVEs
  in modern container base images.
- CVE-2024-5535 (`SSL_select_next_proto`) is the highest-severity CVE in our Phase 1
  set at CRITICAL 9.1. Its memory disclosure aspect (C:H) makes it more than a DoS —
  it is a potential data leakage vector under adversarial conditions.
- All three CVEs have `AC:L` — a reminder that complexity of exploitation is low once
  the attacker is in the HTTPS call path. The Chainguard patch eliminates the
  vulnerable code; the standard image comparison shows what the unpatched surface looks
  like under real load.
